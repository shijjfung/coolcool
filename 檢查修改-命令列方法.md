# 使用命令列檢查修改

## 方法 1：使用命令提示字元（最簡單）

### 步驟：

1. **打開命令提示字元**
   - 按 `Win + R` 鍵
   - 輸入 `cmd`
   - 按 Enter

2. **切換到專案目錄**
   在命令提示字元中輸入：
   ```bash
   cd /d E:\下單庫存管理
   ```
   按 Enter

3. **檢查修改狀態**
   輸入：
   ```bash
   git status
   ```
   按 Enter

4. **查看結果**
   - 如果看到 `pages/api/debug/check-db.ts` → 有修改，需要提交
   - 如果顯示 "nothing to commit" → 沒有修改或已經提交

---

## 方法 2：使用 PowerShell（Windows 10/11）

### 步驟：

1. **打開 PowerShell**
   - 按 `Win + X` 鍵
   - 選擇「Windows PowerShell」或「終端機」
   - 或按 `Win + R`，輸入 `powershell`，按 Enter

2. **切換到專案目錄**
   ```powershell
   cd E:\下單庫存管理
   ```

3. **檢查修改狀態**
   ```powershell
   git status
   ```

---

## 方法 3：直接在檔案總管中打開命令列

### 步驟：

1. **打開檔案總管**
   - 按 `Win + E`

2. **導航到專案資料夾**
   - 前往 `E:\下單庫存管理`

3. **在資料夾中打開命令列**
   - 在資料夾的空白處，按住 `Shift` 鍵
   - 右鍵點擊
   - 選擇「在此處開啟 PowerShell 視窗」或「在此處開啟命令視窗」

4. **檢查修改**
   ```bash
   git status
   ```

---

## 查看修改內容

如果 `git status` 顯示有修改，可以查看具體修改內容：

```bash
git diff pages/api/debug/check-db.ts
```

這會顯示：
- 紅色（前面有 `-`）：被刪除或修改的行
- 綠色（前面有 `+`）：新增的行

---

## 提交修改

如果確認有修改，執行以下命令：

1. **加入修改的檔案**
   ```bash
   git add pages/api/debug/check-db.ts
   ```

2. **提交修改**
   ```bash
   git commit -m "Fix: Make check-db.ts compatible with Vercel/Supabase"
   ```

3. **推送到 GitHub**
   ```bash
   git push origin main
   ```

## 如果沒有安裝 Git

如果執行 `git status` 時出現「'git' 不是內部或外部命令」：

### 檢查是否已安裝 Git：

1. 在命令列輸入：
   ```bash
   git --version
   ```

2. **如果顯示版本號** → Git 已安裝
3. **如果顯示錯誤** → 需要安裝 Git

### 安裝 Git：

1. 下載 Git：https://git-scm.com/download/win
2. 執行安裝程式
3. 安裝完成後重新開啟命令列
