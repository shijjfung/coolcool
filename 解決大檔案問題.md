# 解決 GitHub 大檔案問題

## 問題說明

推送失敗的原因：
- 檔案 `下單庫存管理.7z` 大小為 **179.36 MB**
- GitHub 不允許超過 **100 MB** 的檔案
- 錯誤訊息：`GH001: Large files detected`

## 解決方案

### 方案 1：從 Git 歷史中移除大檔案（推薦）

我已經建立了批次檔：`快速移除大檔案並推送.bat`

**執行步驟**：
1. 雙擊執行 `快速移除大檔案並推送.bat`
2. 批次檔會自動：
   - 從 Git 暫存區移除大檔案
   - 從 Git 歷史中移除大檔案
   - 清理 Git 快取
   - 強制推送到 GitHub

**注意**：這會重寫 Git 歷史，但對於個人專案是安全的。

### 方案 2：手動處理

如果批次檔執行失敗，可以手動執行以下命令：

```cmd
cd /d "E:\下單庫存管理"

REM 1. 從 Git 中移除大檔案
git rm --cached "下單庫存管理.7z"

REM 2. 提交變更
git add .gitignore
git commit -m "Remove large file"

REM 3. 從歷史中移除（可選，但推薦）
git filter-branch --force --index-filter "git rm --cached --ignore-unmatch '下單庫存管理.7z'" --prune-empty --tag-name-filter cat -- --all

REM 4. 清理快取
git reflog expire --expire=now --all
git gc --prune=now --aggressive

REM 5. 強制推送
git push -u origin main --force
```

### 方案 3：刪除並重新建立遠端倉庫（最簡單）

如果上述方法都失敗，可以：

1. **刪除 GitHub 倉庫**：
   - 前往 https://github.com/shijjfung/coolcool/settings
   - 滾動到底部
   - 點擊「Delete this repository」
   - 輸入倉庫名稱確認刪除

2. **重新建立倉庫**：
   - 前往 https://github.com/new
   - 倉庫名稱：`coolcool`
   - 選擇 Public
   - **不要**勾選「Add a README file」
   - 點擊「Create repository」

3. **重新推送**：
   ```cmd
   cd /d "E:\下單庫存管理"
   git remote remove origin
   git remote add origin https://github.com/shijjfung/coolcool.git
   git push -u origin main
   ```

## 已更新的檔案

我已經更新了 `.gitignore`，現在會自動排除：
- `*.7z` - 7z 壓縮檔
- `*.zip` - ZIP 壓縮檔
- `*.rar` - RAR 壓縮檔
- `*.tar.gz` - TAR.GZ 壓縮檔
- `*.iso` - ISO 映像檔
- `*.dmg` - DMG 映像檔
- `*.exe` - 執行檔
- `*.msi` - MSI 安裝檔

## 建議

1. **不要將大檔案加入 Git**：
   - 壓縮檔、安裝檔、映像檔等應該排除
   - 使用 `.gitignore` 自動排除

2. **如果必須儲存大檔案**：
   - 使用 GitHub LFS (Large File Storage)
   - 或使用其他儲存服務（如 Google Drive、Dropbox）

3. **檢查檔案大小**：
   - 在提交前檢查檔案大小
   - GitHub 限制：100 MB 單一檔案
