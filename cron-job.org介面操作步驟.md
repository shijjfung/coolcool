# cron-job.org 介面操作步驟（詳細版）

## 根據您看到的介面

### 步驟 1：設定請求方法

1. 找到 **"請求方法"** 欄位（目前顯示為 `0`）
2. **點擊該欄位**，應該會出現下拉選單
3. 選擇 **`POST`** 或 **`HTTP POST`**
4. 如果沒有下拉選單，可能需要：
   - 點擊旁邊的編輯圖示
   - 或直接輸入 `POST`

### 步驟 2：新增標頭（最重要）

1. 找到 **"標頭"** 區塊（目前顯示「沒有定義自訂標頭」）
2. 在「標頭」區塊中，應該有一個按鈕：
   - **"新增"** 或 **"Add"**
   - **"+"** 按鈕
   - **"定義標頭"** 或 **"Define Headers"**
   - 或是一個可點擊的區域

3. **點擊該按鈕或區域**，會出現新增標頭的介面

4. 新增第一個標頭：
   - **名稱（Name）** 或 **Key**：輸入 `Authorization`
   - **值（Value）**：輸入 `Bearer 你的CRON_SECRET`
     - 例如：`Bearer abc123xyz789`
     - **重要**：`Bearer` 後面必須有一個空格

5. 再次點擊「新增」按鈕，新增第二個標頭：
   - **名稱（Name）**：輸入 `Content-Type`
   - **值（Value）**：輸入 `application/json`

### 步驟 3：檢查「進階」選項

1. 點擊 **"進階"** 按鈕或選項
2. 查看是否有其他需要設定的項目：
   - 請求方法可能也在這裡
   - 標頭設定可能也在這裡

### 步驟 4：其他設定

- **HTTP 身份驗證**：不需要設定，保持空白
- **請求本體**：留空
- **請求逾時**：30 秒（預設即可）
- **時區**：Asia/Taipei（已經正確）

### 步驟 5：儲存

點擊頁面下方或上方的 **"儲存"** 或 **"Save"** 或 **"更新"** 按鈕

## 如果找不到「新增標頭」按鈕

### 方法 A：點擊「標頭」文字本身

有些介面設計，點擊「標頭」這個標題或區域，就會展開設定選項

### 方法 B：查看「進階」選項

1. 點擊 **"進階"** 按鈕
2. 在進階選項中尋找：
   - **"自訂標頭"** 或 **"Custom Headers"**
   - **"HTTP 標頭"** 或 **"HTTP Headers"**
   - **"請求標頭"** 或 **"Request Headers"**

### 方法 C：查看頁面其他位置

- 標頭設定可能在頁面下方
- 或需要先選擇請求方法後才會顯示
- 或需要先儲存一次才能編輯
