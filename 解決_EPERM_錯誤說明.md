# 解決 Next.js EPERM 錯誤說明

## 錯誤訊息
```
[Error: EPERM: operation not permitted, open 'E:\下單庫存管理\.next\trace']
```

## 原因分析

這個錯誤通常發生在 Windows 系統上，可能的原因：

1. **檔案被鎖定**：`.next` 資料夾中的檔案被其他程序（如防毒軟體、檔案總管）鎖定
2. **權限不足**：當前使用者沒有足夠權限訪問 `.next` 資料夾
3. **防毒軟體干擾**：防毒軟體正在掃描或保護該資料夾
4. **檔案系統問題**：NTFS 權限設定問題

## 解決方案

### 方案 1：刪除 .next 資料夾（推薦）

`.next` 資料夾是 Next.js 的建置快取，刪除後會自動重新生成。

**方法 A：使用批次檔**
1. 停止開發伺服器（按 Ctrl+C）
2. 執行 `清理並重新啟動伺服器.bat`

**方法 B：手動刪除**
1. 停止開發伺服器（按 Ctrl+C）
2. 在檔案總管中刪除 `.next` 資料夾
3. 重新執行 `npm run dev`

### 方案 2：以管理員權限執行

1. 右鍵點擊命令提示字元或 PowerShell
2. 選擇「以系統管理員身分執行」
3. 切換到專案目錄
4. 執行 `npm run dev`

### 方案 3：排除防毒軟體掃描

將專案資料夾加入防毒軟體的排除清單：
- Windows Defender：設定 > 病毒與威脅防護 > 排除項目
- 其他防毒軟體：參考其設定選項

### 方案 4：檢查檔案權限

1. 右鍵點擊專案資料夾
2. 選擇「內容」>「安全性」
3. 確認當前使用者有「完全控制」權限

## 注意事項

- **這個錯誤通常不影響功能**：即使出現此錯誤，伺服器通常仍能正常運作
- **`.next` 資料夾可以安全刪除**：這是建置快取，刪除後會自動重新生成
- **建議定期清理**：如果遇到奇怪的建置問題，可以嘗試刪除 `.next` 資料夾

## 快速修復步驟

1. 按 `Ctrl+C` 停止伺服器
2. 執行 `清理並重新啟動伺服器.bat`
3. 如果問題持續，嘗試以管理員權限執行
